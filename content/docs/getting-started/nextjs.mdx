# Next.js Setup

This guide will help you integrate Flint UI into your Next.js application, covering both App Router and Pages Router setups.

## Prerequisites

- Next.js 13+ (App Router) or Next.js 12+ (Pages Router)
- React 18+
- Node.js 16+

## App Router Setup (Next.js 13+)

### 1. Import Global Styles

Add the Flint UI styles to your root layout:

```tsx title="app/layout.tsx"
import '@flint-org/ui/styles';
import './globals.css';

export default function RootLayout({
  children,
}: {
  children: React.ReactNode;
}) {
  return (
    <html lang="en">
      <body>{children}</body>
    </html>
  );
}
```

### 2. Configure Tailwind CSS

Update your `tailwind.config.ts`:

```typescript title="tailwind.config.ts"
import type { Config } from 'tailwindcss';
import flintPreset from '@flint-org/ui/preset';

const config: Config = {
  presets: [flintPreset],
  content: [
    './pages/**/*.{js,ts,jsx,tsx,mdx}',
    './components/**/*.{js,ts,jsx,tsx,mdx}',
    './app/**/*.{js,ts,jsx,tsx,mdx}',
    './node_modules/@flint-org/ui/dist/**/*.js',
  ],
  darkMode: ['class'],
};

export default config;
```

### 3. Use Components

```tsx title="app/page.tsx"
import { Button, Card, CardHeader, CardTitle, CardContent } from '@flint-org/ui';

export default function Home() {
  return (
    <main className="container mx-auto p-8">
      <Card>
        <CardHeader>
          <CardTitle>Welcome to Flint UI</CardTitle>
        </CardHeader>
        <CardContent>
          <p>Get started with beautiful, accessible components.</p>
          <Button className="mt-4">Get Started</Button>
        </CardContent>
      </Card>
    </main>
  );
}
```

## Pages Router Setup (Next.js 12+)

### 1. Import Global Styles

Add the Flint UI styles to your `_app.tsx`:

```tsx title="pages/_app.tsx"
import '@flint-org/ui/styles';
import '../styles/globals.css';
import type { AppProps } from 'next/app';

export default function App({ Component, pageProps }: AppProps) {
  return <Component {...pageProps} />;
}
```

### 2. Configure Tailwind CSS

Same as App Router setup above.

### 3. Use Components

```tsx title="pages/index.tsx"
import { Button, Card, CardHeader, CardTitle, CardContent } from '@flint-org/ui';

export default function Home() {

      </Card>
    </main>
  );
}
```

## Dark Mode Setup

Flint UI includes built-in dark mode support. Install `next-themes`:

```bash
npm install next-themes
```

### App Router

```tsx title="app/providers.tsx"
'use client';

import { ThemeProvider } from 'next-themes';

export function Providers({ children }: { children: React.ReactNode }) {
  return (
    <ThemeProvider attribute="class" defaultTheme="system" enableSystem>
      {children}
    </ThemeProvider>
  );
}
```

```tsx title="app/layout.tsx"
import { Providers } from './providers';
import '@flint-org/ui/styles';

export default function RootLayout({ children }: { children: React.ReactNode }) {
  return (
    <html lang="en" suppressHydrationWarning>
      <body>
        <Providers>{children}</Providers>
      </body>
    </html>
  );
}
```

### Use the Theme Switcher

```tsx
import { ThemeSwitcher } from '@flint-org/ui';

export default function Header() {
  return (
    <header>
      <ThemeSwitcher />
    </header>
  );
}
```

## TypeScript Configuration

Flint UI is fully typed. Ensure your `tsconfig.json` includes:

```json title="tsconfig.json"
{
  "compilerOptions": {
    "strict": true,
    "esModuleInterop": true,
    "skipLibCheck": true,
    "jsx": "preserve"
  }
}
```

## Optimizing Bundle Size

Import only what you need for smaller bundle sizes:

```tsx
// ❌ Imports entire library (~76KB)
import { Button, Card, Input } from '@flint-org/ui';

// ✅ Tree-shakeable imports (~2-5KB per component)
import { Button } from '@flint-org/ui';
import { Card } from '@flint-org/ui';
import { Input } from '@flint-org/ui';
```

## Troubleshooting

### Styles not loading

Make sure you've imported `@flint-org/ui/styles` in your root layout.

### Tailwind classes not working

Ensure `@flint-org/ui/preset` is included in your Tailwind config and the content array includes the Flint UI dist folder.

### TypeScript errors

Run `npm install --save-dev @types/react @types/react-dom` to ensure you have the latest type definitions.
